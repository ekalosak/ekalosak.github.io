Introduction

Central to this report is the following linear mixed model: \[y_{ijkd} = \mu + \delta_i + \pi_j + s_{jk} + e_{ijkd}\] Harville & French proposed this model in 1985 to assess the effects of age and lineage on lamb weight. \(\mu, \delta_i, \textrm{ and } \pi_j\) are fixed effects, and \(s_{jk}\) is a random effect. \(e_{ijkd}\) is additional error. The random effect is \(s_{jk}\sim N(0,G)\) and the error is \(e_{ijdk}\sim N(0,R)\). The fixed effects are age and line (\(\delta \textrm{ and } \pi\)) and the random effect accounts for the sire (\(s\)).

Notice that the data are unbalanced and there are missing observations for some combinations of factors (e.g. Age 2 and Line 4).

Fitting the LMM using ML and REML

## Fit the LMM using both ML and REML

ff = Weight ~ Line + Age + (1 | Sire)
ml_lmm = lmer(formula = ff, data = lamb, REML = F)
reml_lmm = lmer(formula = ff, data = lamb, REML = T)

FIX_reml = fixef(reml_lmm)
RAN_reml = ranef(reml_lmm)$Sire$`(Intercept)`
FIX_ml = fixef(ml_lmm)
RAN_ml = ranef(ml_lmm)$Sire$`(Intercept)`

ml_lmm_nlme = lme(
             fixed = Weight ~ Line + Age,
             random = ~ 1 | Sire,
             data = lamb,
             method = "ML"
            )
reml_lmm_nlme = lme(
             fixed = Weight ~ Line + Age - 1,
             random = ~ 1 | Sire,
             data = lamb,
             method = "REML"
            )

The variance of the random effects (\(s\)) for the model fit using maximum likelihood (ML) is \(\hat{\sigma}_{ML}^{(s)} = 2.0877546\times 10^{-16}\) and that of the restricted maximum likelihood (REML) fit is \(\hat{\sigma}_{REML}^{(s)} = 0.1084728\).

The ranking of sires by their EBLUP random effects measures the quality of each ram. That is, the more positive effect a ram has on the lamb’s weight, the more desirable that ram is as a sire. Using the lme4 package (not shown), the ML random effects EBLUP calculations suffer from underflow - alternatively, using the nlme package as in the table above, the ML and REML estimated rankings appear to agree for the first couple rams, but past that, the rankings diverge. Here it becomes important to clarify that the REML estimates are more accurate - in a situation similar to the Neyman Scott problem, the ML EBLUP using \(\hat{\sigma}_{ML}^{(s)}\) is not consistent, and in fact far underestimates the variance introduced by the random effect. A brief sanity check is possible using the exploratory graph presented in the introduction.

Line-wise means

The line-wise mean EBLUP is defined as follows: \[ \mathcal{L}_i = E[\frac{1}{N_i}\sum_{i=1}^{n_i}\sum_{j=1}^{n_{ij}} y_{ijk} \mid s] \]

The linewise means \(\hat{\mathcal{L}_i}\) are calculated in the analysis script included in the final section of this report. Following those calculations, the lines are ranked as \(4, 1, 3, 5, 2\). Note that the linewise means are identical for the ML and REML estimators. This identity is due to the fact that REML and ML estimations for BLUE’s are identical - the restriction in REML is used for calculating random effect BLUP’s.

Estimating margin of error for sire effects using jackknife

Following the lecture notes (presumably also replicated in Jiang 2007), the mean squared prediction errors for each of the random sire effects is calculated as follows:

\[ mspe(\hat{s_{ij}}) = b_{ij}(\hat{\theta}) - \frac{m-1}{m}\sum_{kl}(b_{ij}(\hat{\theta}_{-kl}) - b_{ij}(\hat{\theta})) + \frac{m-1}{m}\sum_{kl}( \hat{s}_{ij}(\hat{\theta}_{-kl}) - \hat{s}_{ij}(\hat{\theta}) )^2 \]

where \[ b_{ij}(\hat{\theta}) = \frac{\hat{\sigma}_e^2 \hat{\sigma}_s^2} {\hat{\sigma}_e^2 + n_{ij}\hat{\sigma}_s^2} \]

\(n_{ij}\) is the number of measurements from sire \(s_{ij}\), and \(\cdot_{-ij}\) is the calculation performed without the measurements taken from sire \(s_{ij}\). The following plot depicts the random effects \(s_{ij}\) with error bars corresponding to the modified jackknifed MSPE’s.

As is shown in the plot, none of the sire effects are all that significant. Following this result, more replicates are needed to make decisions confidently regarding the rankings of sires. Practically, at this point, all decisions should me made based on line-wise means \(\mathcal{L}_i\).

s = function(sir, ri){
    m = mod_drop(ri)
    ref = ranef(m)$Sire
    w = which(rownames(ref) == sir)
    r = ref$`(Intercept)`[w]
    return(r)
}

b = function(sir, ri=0){
    m = mod_drop(ri)

    r = sig[1]*sig[2]/(sig[1] + nji*sig[2])

}
